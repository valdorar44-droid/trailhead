Product note

Plan on the web. Navigate on the phone.

A browser is a good place to think. It is a questionable place to trust your day when the road gets rough and the signal disappears.

The web planner should feel close to the app, but it has a different job: build the trip, revise the stops, compare options, and prepare the handoff. It should not pretend to be a full offline navigation stack.

What the website should do

Let you plan with a bigger screen, open tabs, a keyboard, and enough context to make better route decisions. It should help you save or send the trip to the phone when the plan is worth keeping.

What the phone should do

Store the route, map packs, and field data locally. Keep location and map controls built for motion. Recover from online/offline weirdness without needing the browser to stay alive.

That split is not a limitation. It is the product being honest about where each surface is strongest.
The planning desk is not the dashboard mount.
